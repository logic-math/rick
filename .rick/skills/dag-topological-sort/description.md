# DAG 拓扑排序技能

## 技能概述

DAG（Directed Acyclic Graph，有向无环图）拓扑排序是一种对有向无环图的顶点进行线性排序的算法，使得对于图中的每一条有向边 (u, v)，顶点 u 都排在顶点 v 之前。

本技能使用 **Kahn 算法** 实现，无需外部依赖，纯 Go 标准库实现。

## 使用场景

### 1. 任务调度系统
- **场景**: 多个任务之间存在依赖关系，需要按正确顺序执行
- **示例**: Rick CLI 的任务执行系统，task1 必须在 task2 之前完成
- **价值**: 自动计算任务执行顺序，避免依赖冲突

### 2. 构建系统
- **场景**: 软件模块之间存在编译依赖
- **示例**: 模块 A 依赖模块 B，B 必须先编译
- **价值**: 优化构建顺序，提高构建效率

### 3. 课程规划
- **场景**: 课程之间有先修关系
- **示例**: 必须先学习"数据结构"才能学习"算法设计"
- **价值**: 生成合理的学习路径

### 4. 数据处理流水线
- **场景**: 数据处理步骤之间有依赖关系
- **示例**: 数据清洗 → 数据转换 → 数据分析
- **价值**: 确保数据处理的正确性

## 核心优势

### ✅ 优点
1. **无外部依赖**: 纯 Go 标准库实现，易于集成
2. **时间复杂度优**: O(V + E)，V 是顶点数，E 是边数
3. **循环检测**: 自动检测并报告循环依赖
4. **易于理解**: Kahn 算法逻辑清晰，易于维护
5. **可扩展性**: 可轻松扩展为并行执行调度

### ⚠️ 注意事项
1. **仅适用于 DAG**: 如果图中有环，算法会报错
2. **不保证唯一性**: 拓扑排序结果可能不唯一（多个合法顺序）
3. **内存占用**: 需要存储整个图结构（入度表、依赖关系）

## 适用条件

- ✅ 任务/节点之间有明确的依赖关系
- ✅ 依赖关系是有向的（A 依赖 B，不代表 B 依赖 A）
- ✅ 依赖关系是无环的（不存在循环依赖）
- ✅ 需要串行或层级并行执行

## 不适用场景

- ❌ 存在循环依赖的场景（需要先解决循环依赖）
- ❌ 无依赖关系的任务（直接并行执行即可）
- ❌ 动态依赖关系（依赖关系在运行时变化）

## 关键概念

### 入度（In-Degree）
- **定义**: 指向某个顶点的边的数量
- **意义**: 入度为 0 的顶点没有依赖，可以优先执行

### 拓扑序（Topological Order）
- **定义**: 满足依赖关系的线性顺序
- **特点**: 对于边 (u, v)，u 必定在 v 之前

### Kahn 算法步骤
1. 计算所有顶点的入度
2. 将入度为 0 的顶点加入队列
3. 从队列中取出顶点，输出到结果
4. 将该顶点的所有邻接顶点的入度减 1
5. 如果邻接顶点的入度变为 0，加入队列
6. 重复步骤 3-5，直到队列为空
7. 如果输出的顶点数小于总顶点数，说明存在环

## 实际效果

在 Rick CLI 项目中：
- **任务依赖**: task1 → task3, task2 → task4, task3 → task5, task4 → task5
- **拓扑排序结果**: [task1, task2, task3, task4, task5]
- **执行顺序**: 按照拓扑排序结果串行执行任务
- **循环检测**: 如果添加 task5 → task1 的依赖，会检测到循环并报错
