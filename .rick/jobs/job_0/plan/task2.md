# 依赖关系
无

# 任务名称
分析代码规范并生成 SPEC.md

# 任务目标
通过分析现有代码库的编码风格、测试规范、Git 提交规范、文档规范，总结并生成项目的开发规范文档 SPEC.md，确保团队协作一致性。

# 关键结果
1. 分析现有代码的编码风格：
   - Go 代码风格（命名、注释、错误处理）
   - 项目结构组织（internal/pkg/cmd 分层）
   - 依赖管理原则（最小化外部依赖）
2. 总结测试规范：
   - 单元测试规范（_test.go 文件组织）
   - 集成测试规范（integration_test.go）
   - 测试覆盖率要求（80%+）
3. 提炼 Git 工作流：
   - 分支管理策略
   - 提交信息格式
   - PR 审查流程
4. 生成 `.rick/SPEC.md` 文件，包含：
   - 代码风格规范
   - 测试规范
   - Git 工作流
   - 文档规范
   - 发布流程

# 测试方法
1. 验证 `.rick/SPEC.md` 文件已创建
2. 检查文件包含以下章节：
   - 代码风格规范（至少 5 条规则）
   - 测试规范（单元测试、集成测试）
   - Git 工作流（分支策略、提交格式）
   - 文档规范（注释、README、Changelog）
   - 发布流程（版本号、打包、发布）
3. 验证规范与现有代码库实践一致
4. 确保规范具有可操作性和可验证性
