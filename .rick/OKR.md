# Rick CLI - OKR (Objectives and Key Results)

> **项目愿景**: 构建业界领先的 Context-First AI Coding Framework，让 AI 编程真正成为人类开发者的生产力倍增器

---

## 📋 项目概述

**项目名称**: Rick CLI
**项目定位**: Context-First AI Coding Framework
**核心理念**: `AICoding = Humans + Agents`，其中 `Agents = Models + Harness`
**技术栈**: Go 1.21+, Cobra, Goldmark, go-git
**当前规模**: 22,724+ 行 Go 代码，8 个核心模块
**版本**: v0.1.0

---

## 🎯 项目愿景（Vision）

Rick CLI 致力于解决 AI 编程工具的核心痛点：**上下文管理**。我们相信，真正有效的 AI 编程不是让 AI 完全自动化编码，而是建立一个**人类主导、AI 辅助**的协作框架。通过精心设计的 Context Loop（plan → doing → learning），Rick 让开发者能够：

1. **精准控制 AI 行为**：通过结构化的任务规划和提示词管理，确保 AI 理解并执行正确的任务
2. **积累项目知识**：通过 learning 阶段，将每次迭代的经验转化为可复用的项目知识库
3. **保持简单高效**：摒弃过度自动化，保留人类决策权，让工具真正服务于开发者而非取代开发者

我们的终极目标是：**让每个开发者都能用 Rick 重构 Rick 本身**，实现工具的自我进化。

## 🎓 核心价值主张（Value Propositions）

### 1. Context-First 设计理念
- **问题**: 传统 AI 编程工具缺乏上下文管理，导致 AI 行为不可控
- **解决方案**: 通过 plan → doing → learning 循环，建立结构化的上下文管理
- **价值**: 让 AI 真正理解项目需求，生成高质量代码

### 2. 人机协作而非自动化
- **问题**: 过度自动化导致开发者失去控制权
- **解决方案**: 人类负责决策（plan、learning），AI 负责执行（doing）
- **价值**: 保持开发者的主导地位，让 AI 成为助手而非替代品

### 3. 知识积累与复用
- **问题**: 每次开发都从零开始，无法积累经验
- **解决方案**: 通过 learning 阶段，将经验转化为可复用的知识库
- **价值**: 项目越做越快，质量越来越高

### 4. 简化设计原则
- **问题**: 复杂的工具增加学习成本和维护负担
- **解决方案**: 最小化依赖，使用标准库，保持简单
- **价值**: 易学易用，易于维护和扩展

### 5. 自我进化能力
- **问题**: 工具无法适应自身的进化需求
- **解决方案**: 支持使用 rick 重构 rick 本身
- **价值**: 工具与项目共同成长，实现真正的自我进化
