## 测试脚本生成要求

1. **覆盖所有功能**: 测试应该覆盖实现的所有功能
2. **边界条件**: 包括边界情况和异常情况的测试
3. **集成测试**: 测试与其他模块的集成
4. **性能测试**: 如果需要，包括性能测试
5. **可读性**: 测试代码应该清晰易读

## 测试用例设计

对于每个功能点，请设计以下类型的测试：

### 1. 正常流程测试
- 验证功能在正常情况下的正确性
- 验证输出格式和数据

### 2. 边界条件测试
- 空输入、null 值
- 最小值、最大值
- 边界附近的值

### 3. 异常情况测试
- 无效输入
- 资源不可用
- 并发访问
- 错误恢复

### 4. 集成测试
- 与依赖模块的集成
- 完整的业务流程

## 输出格式

请生成一个完整的测试脚本文件，包含：

1. **测试文件结构**
   - 导入必要的库
   - 定义测试类或测试函数
   - 设置和清理

2. **测试用例**
   - 每个用例应该有清晰的名称
   - 包含 Arrange-Act-Assert 三个阶段
   - 添加必要的注释

3. **辅助函数**
   - 测试数据生成函数
   - Mock 对象创建
   - 断言辅助函数

4. **运行说明**
   - 如何运行测试
   - 预期结果
   - 测试覆盖率目标

## 质量标准

- 测试覆盖率应该 >= 80%
- 所有测试用例应该能够独立运行
- 测试应该是确定性的（可重复运行得到相同结果）
- 测试执行时间应该在可接受范围内

## 重要提示

1. 测试应该是自动化的，可以被 CI/CD 系统执行
2. 测试失败时应该提供清晰的错误信息
3. 避免测试之间的依赖关系
4. 使用有意义的测试数据
5. 定期运行测试以确保代码质量
