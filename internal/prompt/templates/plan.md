## 规划文档格式

请生成一个结构化的规划文档，包含以下部分：

### 1. 需求分析
- 理解用户需求的核心
- 识别关键功能点
- 分析技术难点

### 2. 任务分解
将需求分解为多个独立的任务（Task），每个任务应该：
- 具有明确的目标
- 有具体的验收标准
- 可以在一个工作周期内完成

### 3. 任务依赖关系
- 分析任务之间的依赖关系
- 确定任务的执行顺序
- 标记关键路径

### 4. 验收标准
为每个任务定义清晰的验收标准，包括：
- 功能完成度
- 代码质量要求
- 测试覆盖率
- 文档要求

## 输出格式

请按照以下 task.md 格式输出每个任务：

```markdown
# 依赖关系
task1, task2, ...

# 任务名称
[任务标题]

# 任务目标
[具体目标描述]

# 关键结果
1. [结果1]
2. [结果2]
...

# 测试方法
1. [测试步骤1]
2. [测试步骤2]
...
```

## 重要提示

1. 每个任务应该相对独立，但可以有依赖关系
2. 任务应该足够具体，使得开发人员能够清楚地理解需求
3. 验收标准应该是可量化的和可验证的
4. 考虑项目的现有架构和技术栈
5. 优先考虑高风险、高价值的任务
